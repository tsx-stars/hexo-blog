# 版本管理

1 git restore <file>//将在工作空间但是不在暂存区的文件撤销更改
2 //将暂存区的文件从暂存区撤出，但不会更改文件git restore ‐‐staged <file> or git reset . or git reset HEAD readme.txt
3 git checkout ‐‐file//可以丢弃工作区的修改
4
5 git diff //查看工作区和版本库的最新版本的区别
6 git [log,reflog] //查看日志
7 git checkout fileName
8 git reset ‐‐hard b498237 //回退到指定版本
9 git reset ‐‐hard HEAD //重置到当前版本
10 git reset ‐‐hard HEADA~ //重置到上个版本
11 git reset ‐‐hard HEAD~ 2  //重置到往上 2 个版本
12 git reset ‐‐hard HEAD@{版本号} //七位版本号重置到指定版本
13 有了参数‐‐hard,直接把工作区的内容也修改了，不加‐‐hard的时候只是操作了暂存区
14 git rev‐parse ‐‐short HEAD //将要回退版本的hash
15 连退两步。HEAD~ 后面加数字表示后退的步数，每次后退都默认退到第一个父提交上，HEAD~ 2  表示
16 退一步到第二个父提交上。HEAD^ 后面加数字表示只退一步，但是这一步后退到数字表示的父提交上，HEAD^ 2  表示
17 git 在查看多分支提交记录时，日志的先后顺序不代表提交时间的先后顺序。
18 git reset 命令是一个重置 HEAD 的命令，可以指挥版本库指向任何一个合法提交。
19
20 git checkout  09 bd8cc1 hello.txt//回滚
21
22 舍弃本地代码,远端版本覆盖本地版本
23 git fetch ‐‐all
24 git reset ‐‐hard origin/master
25 git fetch

27 有文件未提交时切分支
28 git clean n //这个是清除文件预览
29 git clean ‐f //强制清除文件

# 分支

1 git branch //查看本地分支
2 git branch ‐r //查看远程分支
3 git branch ‐a //列出本地分支和远程分支
4 git branch ‐vv //分支与远程对应关系
5
6 git branch dev //创建分支
7 git push origin dev //分支推送到远程
8 ev分支，并切换到本地的git checkout ‐b dev origindev分支 /dev，//作用是checkout远程的dev分支，在本地起名为d
9 git checkout dev //切换分支
10 git branch ‐d | ‐D dev //删除分支，D表示强制删除
11 git push origin ‐‐delete dev //删除远程分支
12 git merge origin/dev ‐m '合并分支'
13
14 git remote show origin //查看信息

# 常用命令
